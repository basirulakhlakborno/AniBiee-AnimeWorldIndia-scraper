## 2025-10-24 - Prevent dead-end user flows on error pages
**Learning:** Error pages (like 403 and 404) without explicit navigation paths create dead-ends for users, leading to confusion and abandonment.
**Action:** Always include an explicit "Return to Homepage" (or similar) interaction path on error pages, styled as a prominent secondary action (e.g., glassmorphism on dark themes). Ensure keyboard accessibility with focus states.
